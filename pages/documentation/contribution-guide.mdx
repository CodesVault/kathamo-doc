Kathamo is build in a component based way. This means that you can use the components you need and leave the rest out.
Each component is a [composer package](https://packagist.org/packages/codesvault/), and all components are developed by the same team.

<br />

## Contribution process

Create a Github issue for a feature request or bug report. Before creating the issue please check if it already exists.
Then fork the repository, create a new branch with relevant name and make your changes. When you are done with your changes create a pull request.

After merging a pull request; issue creator, PR contributors, maintainers and documentation writers will be added to the contributors list.
